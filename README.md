# CS-330
How do I approach designing software?
-I like to look at what the main objectives of the project that I am working on are first before just diving into it. 
The final project of CS-330 had an objective of not only being able to render certain objects, but to create a cohesive scene that mimicked our example image. 

What new design skills has your work on the project helped you to craft?
-I found myself thinking outside of the box and using a more visual process than I normally do. I would add code and then run the program to see how the scene was setting up.
I would do this several times with one object before moving onto the next because one wrong move could make the whole scene feel off. This was the case for not only the object,
but majority of the components like lighting and scaling too. 
-Another key take away that I got from this project was learning that without certain materials on the objects, the lighting would not work how I wanted it. I kept getting a really
dark image before I added a shader material to my desktop, but once I added it, the light adjusted the perfect way and it all made sense. 

What design process did you follow for your project work?
-First I decided what I wanted to incorporate in my scene. I wanted something that felt personal, but that wasn't too difficult to render for my first time using openGL. Then I started
out by creating the notebook in my scene. From there I actually stopped creating objects and went on to add texture, shader materials and lighting to the image. The last steps were to add
the final objects in the scene and adjust anything else as needed. I realize that I maybe should have created all objects first and then moved forward, but it ended up working out. 

How do I approach developing programs?
-I tend to work in smaller parts. Breaking down a to-do list into easier to finish sections makes sure that everything gets completed to the best possible outcome. 

What new development strategies did you use while working on your 3D scene?
-I think the biggest thing goes back to my lighting issue. I was having troubles with a dark light, no matter what I did, so I backtracked and went to the last thing I worked on. This happend
to be shader materials so I played around with the objects and their materials until I found a solution. 

How did iteration factor into your development?
-Everything in my project required so many adjustments. I had to adjust the objects (placement and size) and the lighting of the scene so much. I actually got frustrated at some points because
it just felt so tedious. It just goes to show that with some refinement, that small changes can lead to great outcomes. 

How has your approach to developing code evolved throughout the milestones, which led you to the project’s completion?
-I always thought that it was just about making the code work properly, which it is, but there is more to it! I learned throughout that even though I had proper code lines written, that maybe they
weren't in the right place so it wasn't producing the proper outcomes. 

How can computer science help me in reaching my goals?
-I am continuously learning about things other than just software development. I think that without learning about certain processes and building upon my problem solving skills that I would be in 
a much different place to where I am now. 

How do computational graphics and visualizations give you new knowledge and skills that can be applied in your future educational pathway?
-Understanding computational graphics and visualizations has helped me expand on my knowledge about math fields like vectors and transformations. It has also helped me to understand that it takes 
a lot of work to make things cohesive, but if you take your time and do it properly that it will be worth it. This was a big help in building my patience and problem solving skills.
